Var vs Let vs Const What to use ?

# Before ES6 var was the only way to declare variables.

# ECMAScript 6 introduced let & const.

Var

● var scope is global when a var variable is declared outside a function. This means that any variable that is declared with var outside a function block is available for use in the whole window.

● var is function scoped when it is declared within a function. This means that it is available only within that function.

Javascript 

var greeting = "say hi";

function newFunction() {
    var hello = "hello";
}

console.log(greeting); // say hi

console.log(hello); // Uncaught ReferenceError: hello is not defined

● Here, greeting is globally scoped because it exists outside of function while hello is function scoped. So we cannot access the variable hello outside of a function.

● var variables can be re-declared and updated also.

Problem with var
Javascript 

var greeting = "hey hi";
var times = 4;

if (times > 3) {
    var greeting = "say Hello instead";
}

console.log(greeting) // "say Hello instead"
Hoisting of Var

● Hoisting is a JavaScript mechanism where variables and function declarations are moved to the top of their scope before span execution.

Let

● let is block lvl scoped

● So a variable declared in a block with let is only available for use within that block.

Javascript 

let greeting = "say Hi";
let times = 4;

if (times > 3) {
    let hello = "say Hello instead";;
    console.log(hello); // "say Hello instead"
}

console.log(hello) // hello is not defined

// We see that using hello outside its block (the
// curly braces where it was defined) returns an error.

// This is because let variables are block scoped

● let can be updated but not re-declared.

Javascript 

let greeting = "say Hi";
greeting = "say Hello instead";
// This is fine we are updating its value.
Javascript 

// This will give an error.
let greeting = "say Hi";
let greeting = "say Hello instead";
// error: Identifier 'greeting' has already been declared
Javascript 

// However, if the same variable is defined in
// different scopes, there will be no error:

let greeting = "say Hi";
if (true) {
    let greeting = "say Hello instead";
    console.log(greeting); // "say Hello instead"
}
console.log(greeting); // "say Hi"
Const

● const declarations are also block level scoped.

● Like let declarations, const declarations can only be accessed within the block they were declared.

● const cannot be updated or re-declared

Javascript 

const greeting = "say Hi";
greeting = "say Hello instead";
// error: Assignment to constant variable.
Javascript 

const greeting = "say Hi"
const greeting = "say Hello instead"; 
// error: Identifier ‘greeting' has already been declared
Javascript 

const greeting = {
    message: "say Hi",
    times: 4
}

greeting = {
    words: "Hello",
    number: "five"
} 
// error: Assignment to constant variable. 

greeting.message = "say Hello instead";
// But the properties of objects can be updated.
Summary

● var declarations are globally scoped or function scoped while let and const are block scoped.

● var variables can be updated and re-declared within its scope; let variables can be updated but not re-declared; const variables can neither be updated nor re-declared.

● They are all hoisted to the top of their scope. But while var variables are initialized with undefined, let and const variables are not initialized.

● While var and let can be declared without being initialized, const must be initialized during declaration.

Note - Avoid using var as much as you can,use let when you want to update value and use const when we dont want to update/reassign value.
